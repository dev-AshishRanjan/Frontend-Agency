# Brand Discovery

Use this skill to turn an unclear brief into decision-ready brand, audience, and positioning context.

## When to Run

Run before any frontend strategy, design, copy, or implementation work when the brief is broad, greenfield, or missing audience, positioning, or constraint context.

## Workflow

1. Identify the business goal, primary audience, product category, offer, and desired action.
2. Extract brand attributes from provided copy, assets, examples, competitors, and constraints.
3. Ask only blocking questions. If a detail is absent but non-blocking, state a professional assumption.
4. Define audience pains, desired outcomes, objections, and trust signals.
5. Produce a discovery brief that downstream skills can use without reinterpreting the original request.

## Output

Return:

- **Brand summary** — name, category, positioning in one sentence
- **Audience segments** — prioritized list with primary user defined
- **Positioning hypothesis** — differentiation and value proposition
- **Conversion goal** — desired user action and success metric
- **Content and proof requirements** — testimonials, data, demos needed
- **Constraints and assumptions** — technical, brand, timeline, scope
- **Risks** — what could make the design generic or inaccurate
